# Audit Playbook

这不是执行脚本，是思维框架参考。具体怎么审计、先读哪里、花多少时间——由你根据当前 finding 自主决定。

## 代码阅读策略

从哪里开始不取决于惯例，取决于已有线索。常见的有效切入点：

**从 evidence.source 已知时**：直接找到 source 所在文件，沿调用链向下追踪，不要先读无关代码。

**从漏洞类别已知时**：直接在代码库中搜索该类漏洞的典型 sink 函数（如 `readFileSync`、`exec`、`query`），找到 sink 后再向上追溯调用者，看 source 是否可控。

**从 package.json `main` 出发**：找到模块的公开 API 入口，识别哪些参数来自外部调用方。用 `resolve_source_path.py` 直接得到 raw URL，省去路径猜测。

**关键原则**：读 2–3 个相关文件比读 10 个无关文件更有价值。找到 source→sink 的具体路径后立即记录，不要继续扩大阅读范围。

## Dataflow 分析框架

三个问题，按顺序回答：

**① Source 是什么？**  
攻击者能控制哪个输入？控制程度如何（完全控制 / 部分控制 / 只能影响某个字段）？这个输入经过哪个函数进入系统？

**② Sink 在哪？**  
危险操作在哪一行？它接收的参数来自哪里？中间经过了几层函数传递？每一层传递是完整传递还是有提取/变换？

**③ 中间有没有 sanitizer？**  
每一层传递处，有没有对输入做检查或变换？如果有，这个检查能否覆盖所有危险输入形式？

答完这三个问题，你就有了 source/sink/guard 的具体内容。如果任意一个问题无法确定，置信度就不是 high。

## Guard 评估框架

发现疑似 guard 时，区分三种情况：

**Guard 不存在**：从 source 到 sink 没有任何检查，直接可利用。这是最简单的情况，直接填写 `evidence.guard: "缺失——无任何路径检查"`。

**Guard 存在但可绕过**：有检查，但存在漏洞。常见模式：
- 编码绕过：检查明文路径但 sink 解码后处理（`../` vs `%2e%2e%2f`）
- 不完整覆盖：只检查了某些前缀但没有 normalize（`/safe/../../../etc/passwd`）
- 时序问题：检查和使用之间有状态变化

填写 `evidence.guard: "存在但可绕过——[具体绕过方式]"`。

**Guard 有效**：能证明任意输入都无法到达 sink，或到达后无法造成危险。此时应诚实标 blocked，说明 guard 有效。

## 置信度标准

**High**：source 和 sink 都有精确的 `file:line` 引用，数据流连续可追踪，guard 状态明确。

**Medium**：source 或 sink 有一个是推断得出而非直接确认，或数据流中有一段无法直接读到代码（如依赖关系跨包）。

**Low**：仅凭关键字匹配或模式推断，没有实际代码路径支撑。Low 置信度不足以支撑 confirmed 结论。

## 何时停止调查

**升为 confirmed 的条件**：source/sink/guard 全部 high 或 medium 置信度，且能描述可本地复现的步骤。不需要等到"100% 确定"——research 质量，不是法庭标准。

**标为 blocked 而非继续调查的时机**：
- 源文件不可访问（私有仓库、fetch budget 耗尽）
- guard 经过认真分析后判定有效
- 发现高度疑似的已有 CVE（优先标 blocked，让去重确认）
- 触发条件只有开发者权限才能满足（不满足攻击者视角）
- 投入超过合理时间仍无法建立完整数据流

不要为了避免 blocked 而降低证据标准。blocked 是有价值的研究结果，不是失败。

## PoC 描述规范

`evidence.reproducer` 应该让另一个研究员能够在本地重现，不依赖你的上下文。包含：

1. **环境**：哪个版本，哪个语言运行时
2. **输入构造**：具体的输入内容（文件内容、API 调用参数等）
3. **触发路径**：调用哪个函数/接口
4. **预期观测**：成功利用时应该看到什么（报错信息、文件内容、命令输出）

示例格式：
```
版本：markdown-it-include@2.0.0，Node.js
输入：包含 !!!include(../../../../etc/passwd)!!! 的 markdown 文件
触发：调用 md.render(input, {root: '/tmp/safe'})
预期：输出中包含 /etc/passwd 的文件内容，或抛出包含该路径的读取错误
```
